wma_pipeline.py — Architecture & Methodologie
Les anomalies de la substance blanche sont frequentes mais sous-detectees
  • ~500 sujets WMA+ vs ~6000 WMA- dans ABCD
  • Prevalence < 8% — desequilibre severe
  • Detection manuelle chronophage et variable
  • Besoin d'un outil automatise fiable

Cohorte ABCD

Adolescent Brain Cognitive Development Study — plus grande etude longitudinale du developpement cerebral aux USA. Donnees IRM multi-sites avec labels radiologiques.

WMA (White Matter Abnormality)

Hyperintensites de la substance blanche visibles en IRM, marqueurs de leucoaraiose pouvant indiquer des pathologies neurovasculaires.

Le defi : classifier sans FLAIR, avec un ratio positif/negatif de 1:12
  • Pas de sequence FLAIR dans ABCD — modalite de reference pour les WMA
  • Seulement T1w + T2w disponibles → signal plus subtil
  • Desequilibre extreme : metriques classiques (accuracy, AUC) sont trompeuses
  • Metrique cible : AUPREC (Area Under Precision-Recall Curve)
La metrique AUPREC est plus informative que l'AUROC en cas de forte prevalence negative — elle penalise les faux positifs dans un contexte rare-positif.

Etape 3
Preprocessing & Augmentation
L'entree 3 canaux T1 + T2 + ratio T2/T1 compense l'absence de FLAIR

Canal 1 : T1w

Contraste anatomique. Substance blanche brillante, grise sombre.

Canal 2 : T2w

Contraste complementaire. Les lesions WMA apparaissent en hyperintensites.

Canal 3 : T2/T1

Ratio synthetique. Amplifie le contraste des anomalies — proxy de FLAIR.

ratio = T2 / (|T1| + eps) # eps = 1e-3 ratio = clamp(ratio, 0, 5) # eviter les extremes ratio = (ratio - mean) / (std + eps) # normalise sur le masque T1 > 0.01

Design 80/20

Maximiser l'impact sur AUPREC avec le minimum de complexite. ResNet-10 est le choix delibere : plus compact, plus stable, plus rapide a entrainer.

SwinUNETR reste une option

Disponible via --backbone swin_tiny pour benchmarking. Utilise des checkpoints gradient pour la memoire.

Stochastic Depth (Drop Path)

Dropout3d applique entre les stages du ResNet. Regularise les feature maps intermediaires, pas seulement la tete.

Progressive Unfreezing

Backbone gele pendant les freeze_epochs premieres epoques (defaut: 5). Seule la tete apprend d'abord.

Etape 5
Entrainement 2 stages
Stage 1 (CE+Focal) stabilise les scores, Stage 2 (APLoss) optimise directement AUPREC

Stage 1 : Warmup

Loss : FocalBCE
(gamma=2, alpha=0.85)

Optimizer : AdamW

Objectif : Apprendre une distribution de scores bien separee

Epoques 1 → 15

Stage 2 : Fine-tune

Loss : APLoss (LibAUC)
(gamma=0.9, margin=1.0)

Optimizer : SOAP

Objectif : Maximiser directement l'AP

Epoques 16 → 40
La transition se fait a l'epoque warmup_epochs + 1. Si LibAUC n'est pas installe, Stage 2 continue avec la loss CE.
FocalBCE surpondere les positifs difficiles avec alpha=0.85 et gamma=2
FocalBCE(logits, targets): bce = BCE(logits, targets) # loss standard pt = p * t + (1-p) * (1-t) # probabilite du vrai label alpha_t = 0.85*t + 0.15*(1-t) # poids de classe focal = alpha_t * (1-pt)^2 * bce return bce.mean() + 0.5 * focal.mean()

Pourquoi combiner BCE + Focal ?

  • BCE pur : gradient stable mais ignore le desequilibre
  • Focal pur : peut rendre l'apprentissage instable tot dans le training
  • Somme ponderee : stabilite + attention aux cas rares
APLoss optimise l'aire sous la courbe PR de facon differentiable

APLoss (LibAUC)

  • Approximation surrogat de l'Average Precision
  • gamma=0.9 : facteur de momentum
  • margin=1.0 : marge entre paires +/-
  • Necessite un DualSampler pour equilibrer les batchs

SOAP Optimizer

  • Optimizer dedie aux losses non-decomposables
  • lr = lr_base / 3 pour la stabilite
  • epoch_decay=1e-3 : regularisation interne

DualSampler

Garantit au moins 1 positif par batch. Essentiel pour que les paires +/- soient toujours disponibles dans la loss.

Layer-wise LR decay, EMA et cosine schedule stabilisent le fine-tuning

Layer-wise LR Decay

  • Facteur 0.75 par couche
  • Couches profondes : LR tres bas
  • Tete : LR complet
  • Backbone factor : 0.1x

layer4: 3e-5
layer3: 2.25e-5
layer2: 1.7e-5
head: 3e-4

EMA (Exponential Moving Average)

  • Decay = 0.9995
  • Moyenne glissante des poids
  • Modele EMA utilise pour la validation
  • Sauvegarde dans le checkpoint

Cosine Annealing

  • LR suit un cosinus decroissant
  • Reset au passage Stage 1 → 2
  • Plancher a 0.1% du LR initial
  • Un schedule par stage
bf16 + gradient accumulation 4x simulent un batch effectif de 16 sur GPU limitee

Mixed Precision (bf16)

  • torch.amp.autocast("cuda", dtype=bfloat16)
  • Forward pass en bf16, loss en float32
  • Reduit la memoire GPU ~2x
  • bfloat16 > float16 : pas besoin de loss scaling

Gradient Accumulation

  • Batch reel : 4
  • Accumulation : 4 steps
  • Batch effectif : 16
  • Clip grad norm a 1.0

Early Stopping

Patience = 10 epoques sans amelioration de val AUPREC (> min_delta 0.001). Reset a chaque changement de stage.

Etape 6
Validation & Evaluation
StratifiedGroupKFold empeche la fuite de donnees entre folds
  • 5 folds par defaut (configurable)
  • Stratifie : meme ratio WMA+/WMA- dans chaque fold
  • Groupe = subject_id : toutes les sessions d'un sujet dans le meme fold
  • Empeche l'apprentissage de l'anatomie individuelle

Pourquoi GroupKFold ?

Un sujet peut avoir plusieurs sessions (ses-00A, ses-02A...). Sans groupement, le modele pourrait "reconnoaitre" un cerveau deja vu → surestimation des performances.

Alerte de performance

Si val AUPREC < 0.18 apres Stage 1, un warning est emis. Cela indique un probleme potentiel (donnees, hyperparametres, ou convergence).

L'evaluation rapporte AUPREC, AUROC, Brier, F1 optimal et Sens@Spec90

Seuil optimal

Calcule sur la courbe PR en maximisant le F1-score. Pas un seuil arbitraire a 0.5 — adapte au desequilibre de classes.

TTA (Test-Time Augmentation)

Moyenne des predictions sur 4 versions : originale + 3 flips axiaux. Active via --use_tta. Ameliore la robustesse de l'inference.
